# B2 implementation plan — virtual memory & address spaces

> **Status:** Active. Decisions locked 2026-05-20. See [`../roadmap.md`](../roadmap.md) Track B → B2.

## Goal
Physical frame allocator + 4-level paging, demand-paging via a `PageFaultHandler` HSM, and per-process address-space machinery (consumed at B3). Page faults are recovered or cleanly fatal-ed, never a silent triple-fault.

## Locked decisions
1. **Bitmap frame allocator** (simplest; no contiguous-alloc need at B2).
2. **Extend Limine's HHDM + existing page tables** for Steps 1–3 (lower triple-fault risk); build fresh PML4s only at Step 4 for new address spaces.
3. **`$InitMemory` HSM phase becomes real at B2** — it runs frame-allocator/paging init instead of printing a stub. (`$InitIDT`/`$InitTimer` follow later.) Note: the heap (`linked_list_allocator`) stays initialized in `kmain` before `Kernel::__create()`, since the HSM constructor itself allocates; `$InitMemory` initializes the *physical frame* allocator + paging on top.
4. **Build address-space machinery + a smoke test at B2** so B3 consumes a tested primitive.

## Native/Frame split (B2 is ~85% native)
- **Native:** physical frame allocator (over Limine's memory map), 4-level page-table walk/build (`map`/`unmap`/`translate`), physical access via Limine HHDM, the `#PF` (vector 14) stub (reads CR2 + error code), address-space construction (new PML4 + copy kernel higher-half) + CR3 switch.
- **Frame (`PageFaultHandler`):** classify a fault → dispatch. Synchronous exception + `#PF` gate clears IF ⇒ no preemption, no concurrent fault (nested = double-fault, caught) ⇒ the stub drives one global instance **synchronously, no critical section, no queue**. A clean Frame-in-kernel fit.

## `PageFaultHandler` HSM
- `$Classifying` (entry) → `$LazyFault | $Fatal` (exercised at B2); `$StackGrow | $CopyOnWrite` declared, wired at B3/B4.
- `$LazyFault`: addr in a registered demand region → alloc frame, map, retry. Out of frames → `=> $^` to parent → `$Fatal`.
- `$Fatal`: print + halt (B2); kill process (B3).
- Parent `$FaultActive` owns the unrecoverable handler (the `=> $^` target).

## Incremental steps (QEMU-validated each)
1. **Physical frame allocator** — Limine memory map + HHDM requests; bitmap; `alloc_frame`/`free_frame`. Smoke: alloc/free/realloc sanity. (B2-3)
2. **Paging** — `map`/`unmap`/`translate` extending Limine's tables, frames via HHDM. Smoke: map → write pattern → read back → unmap. (B2-4)
3. **`#PF` + `PageFaultHandler`** — host-test classification; wire vector 14 stub (CR2 + error code) → HSM. Smoke: demand region faults in (`$LazyFault`, access then succeeds); unmapped access is `$Fatal` (prints, no triple-fault). (B2-5)
4. **Address spaces** — new PML4 (copy kernel higher-half) + CR3 switch. Smoke: create 2nd space, map, switch, verify, switch back.
5. **`$InitMemory` real** — fold Step 1–2 init into the HSM phase; per-system doc for `PageFaultHandler`; diagrams. (B2-6)

## Expected framec gates
HSM forwarding (`=> $^` from `$LazyFault` on OOM); transition guards (classification predicates); driving a Frame HSM from a synchronous exception stub (expected fine — IF=0).

## Risks
- Paging (#1): wrong PTE → triple-fault. Mitigated by extending Limine's working tables + HHDM and the `#PF`/double-fault safety net.
- The `#PF` stub: error code on the stack + CR2 read; wrong frame layout faults the handler.

## Test mapping
B2-1/B2-2 (PageFaultHandler snapshot + behavioral) → Step 3 (host). B2-3/B2-4/B2-5 → Steps 1–3 (QEMU). B2-6/B2-7 (docs, gates) → Step 5 / throughout.
