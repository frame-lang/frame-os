# B1 implementation plan — preemptive multitasking

> **Status:** Active. Decisions locked 2026-05-20. This plan supersedes the
> earlier cooperative B1 sketch (the roadmap re-baseline made B1
> preemptive). See [`../roadmap.md`](../roadmap.md) Track B → B1.

## Goal

Multiple kernel threads, preempted by a periodic timer interrupt: a thread
in a tight loop still yields the CPU to others. This is where the kernel
becomes interrupt-driven and "feels like a kernel."

## The crux — native/Frame split (locked)

Preemption is **mostly native**. The switch *must* run inside the timer
ISR (a tight-loop thread never reaches any other safe point), so the ISR
saves full register state, picks the next thread from a **native
ready-queue**, swaps stacks, and `iretq`s — it **never calls a Frame
system** (Frame dispatch is non-reentrant). The Frame `Scheduler` and
`Task` are touched **only from normal context** (admit/block/unblock/exit),
behind a short ISR-safe lock on the shared ready-queue.

```
  timer ISR (native)          normal context             shared (native, lock)
  ─────────────────           ──────────────             ─────────────────────
  save full regs              Scheduler events:          ready-queue: [tid,...]
  pick next (round-robin)       task_ready/task_unready  current_thread: tid
  swap stack, iretq           Task lifecycle:
       │                        make_ready/block/
       └─ reads ──────────────  unblock/terminate
                                    │
                              mutates (with lock; ISR also takes it)
```

### Locked decisions

1. **The deferred-event queue moves to B4, not B1.** Preemption can't be
   driven through a drained queue (the tight-loop thread never returns to
   the main loop to drain it), so the ISR switches natively and needs no
   queue. The queue's first hard requirement is a **device-completion
   interrupt** (disk at B4) that must deliver an event into a possibly-in-
   flight Frame system. Roadmap amended accordingly.
2. **`Task` has no `$Running` state.** "Currently on the CPU" flips every
   tick and would have to fire from the ISR (forbidden) — it's native
   (`current_thread`). `Task` models the coarse lifecycle:
   `$Created → $Ready ⇄ $Blocked → $Terminated`.
3. **`Scheduler` ships small: `$Idle` / `$Active`.** Picking and switching
   are native ISR work, so the only genuinely-different-behavior states are
   "nothing runnable → `hlt`" vs "keep switching." Accepted as honest-
   minimal (same call as SerialDriver); reassessed at B3 when blocking/
   waiting/zombie give it real states.

## Frame systems

**`Task(id: u32)`** — `frame/task.frs`
- `$Created` → `make_ready` → `$Ready`; `terminate` → `$Terminated`
- `$Ready` → `block` → `$Blocked`; `terminate` → `$Terminated`
- `$Blocked` → `unblock` → `$Ready`; `terminate` → `$Terminated`
- `$Terminated` — sink
- queries: `id()`, `is_ready()`, `is_blocked()`, `is_terminated()` (defaults + per-state overrides)

**`Scheduler`** — `frame/scheduler.frs`
- domain `runnable: u32`
- `$Idle` → `task_ready` (`runnable += 1`) → `$Active`; `is_idle()=true`
- `$Active` → `task_ready` (`runnable += 1`, stay); `task_unready` (`runnable -= 1`; → `$Idle` when 0)
- query: `is_idle()`, `runnable_count()`

## Native components (later steps)

- IDT + exception handlers (faults print + halt). Reuse Limine's GDT; TSS deferred to B3.
- 8259 PIC remap + PIT channel 0 periodic (~100 Hz).
- Preemptive context switch: full register save/restore from interrupt context; per-task 16 KiB static kernel stacks; fresh-task stack-frame crafting.
- Native ready-queue + `current_thread` behind a short interrupt-safe lock.

## Incremental steps (each leaves a green tree)

1. **Frame layer, host-tested (this step).** `Task` + `Scheduler` FSMs;
   behavioral + snapshot tests in `kernel-tests`. No bare metal. De-risks
   the (small) Frame part and locks the state graphs.
2. **Native context switch, no timer.** Two threads cooperatively calling
   `switch()` (print A → switch → print B). QEMU smoke asserts interleave.
   Isolates the #1-risk asm.
3. **IDT + PIT + preemption.** Timer ISR does the switch. QEMU smoke: a
   tight-loop thread *is* preempted. `$InitIDT`/`$InitTimer` become real.
4. **Multi-thread demo + `$Idle` halt.** N threads emit markers; `Scheduler`
   halts in `$Idle` when none runnable; per-system docs + diagrams.

## Risks

- **Context-switch asm is #1** — wrong stack-frame layout → triple fault. Step 2 isolates it.
- **GDT/IDT** — confirm Limine's GDT code-selector works for IDT gates.
- **The ready-queue lock** — first ISR-vs-normal-context concurrency; small now, real.
- **`Scheduler` thinness** — accepted (decision 3); will not be padded.
