# B3 implementation plan — user mode, processes, syscalls, ELF, fork/exec

> **Status:** Active. Decisions locked 2026-05-20. See [`../roadmap.md`](../roadmap.md) Track B → B3. The xv6-bar milestone; "very large" (effectively 5 sub-milestones).

## Goal
The user/kernel boundary. Ring-3 execution, `syscall`/`sysret`, per-process address spaces, ELF loading, `fork`/`exec`, basic signals. A real user program runs, hardware-isolated, and makes syscalls.

## Locked decisions
1. **`syscall`/`sysret`** (modern x86_64 fast path), not legacy `int 0x80`.
2. **`fork` = eager full copy** at B3; **COW as a follow-on** (activates `PageFaultHandler`'s declared `$CopyOnWrite`).
3. **User programs = freestanding Rust, raw syscalls, static ELF baked into the kernel image** (defer a libc/crt0).
4. **`Process` is the scheduled unit** (supersedes the B1 kernel-thread demo); `Task`'s lifecycle shape carries into `Process`.
5. **Single-core simplification:** the syscall entry switches stacks via static `KERNEL_RSP`/`USER_RSP` (no swapgs/GS). Per-CPU GS arrives at B7 (SMP).

## Native/Frame split
- **Native:** GDT (null/kcode/kdata/ucode/udata) + TSS (RSP0); `syscall`/`sysret` MSRs (STAR/LSTAR/SFMASK/EFER.SCE) + entry stub; ring-3 entry (`iretq`) + return-to-kernel longjmp; ELF byte parsing + segment mapping; `fork`/`exec`; user-program toolchain; scheduler extension (CR3 per process).
- **Frame (largest payload yet; finally exercises `=> $^`):**
  - `Process` — HSM `$Created → $Ready → $Running → $Blocked → $Zombie → $Reaped` (load-bearing: syscalls + scheduler drive it; state-dependent `kill()`).
  - `ProcessTable` — slot manager (`$Empty → reserve → active → zombie-awaiting-reap → free`).
  - `SyscallDispatcher` — HSM `$Validating → $Executing` under `$Active`; children forward `bad-arg`/`permission`/`OOM` via `=> $^`.
  - `ElfLoader` — `$ReadingHeader → $ValidatingHeader → $MappingSegments → $BuildingStack → $Done`; `$Failed` cleans up partial mappings.

## Incremental steps (QEMU-validated each)
1. **Ring 3 + `syscall`/`sysret` round-trip** (native).
   - 1a: own GDT + TSS, `lgdt`/reload segments/`ltr`; kernel survives. 
   - 1b: syscall MSRs + entry stub + `enter_user`/longjmp + a hand-crafted ring-3 blob doing `write` + `exit`. Proves the boundary.
2. **`SyscallDispatcher` HSM** — route the syscall entry through it; host-test validate→execute + `=> $^` error forwarding.
3. **`Process` + `ProcessTable`** — lifecycles as Frame; extend the scheduler to schedule processes (CR3 switch); `exit`→`$Zombie`, `wait`→`$Reaped`.
4. **`ElfLoader` + a real user program** — load a baked freestanding ELF, run in ring 3; prove isolation (user reads kernel mem → `$Fatal` kills the *process*, not the kernel); `$Failed` on a corrupt ELF.
5. **`fork`/`exec` + basic signals** — `fork` (eager copy), `exec`, `SIGKILL`/`SIGSEGV`/`SIGCHLD`.

## Expected framec gates
`=> $^` forwarding (finally load-bearing); scale (one `Process` per process — `Rc`/`Vec`/`BTreeMap`); HSM depth; `ElfLoader` `$Failed` cleanup.

## Risks
- GDT/TSS/syscall asm (#1: triple-fault/#GP); ring transitions + stack switching; the longjmp; the user toolchain (baking a static ELF); scope (5 sub-milestones).
- Mitigation: incremental sub-steps, QEMU-validate each, lean on the `#PF`/exception safety net for diagnostics.

## Test mapping
B3-1..B3-4 (Frame snapshots + behavioral) → Steps 2–3 (host). B3-5/6/7 (user-mode runs, isolation, fork/exec) → Steps 4–5 (QEMU). Syscall ABI doc + per-system docs → throughout.
